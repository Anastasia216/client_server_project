# client-server-project
Учасники команди: Козинська Анастасія Олександрівна та Карнаух Дарія Андріївна.
Тема проєкту: Онлайн чат (спрощений клон телеграму/whats app) , який працює в реальному часі.

**Запланований функціонал:**
1. Авторизація та реєстрація
2. Обмін повідомленнями (надсилання повідомлень у реальому часі в приватні та групові чати, збереження історії повідомлень, відображення часу надсилання та статусу доставки повідомлення)
3. Робота з чатами (створювати, переглядати та видаляти чати, додавати учасників у чат та переглядати їх та виходити з чатів)
4. Передача файлів (підтримка надсилання зображень й документів та збереження їх на сервері)
5. Статуси користувачів (статус онлайн/офлайн та час останньої активності)
6. Можливості адміністратора (блокувати користувачів , видаляти повідомлення та чати, доступ до перегляду серверних логів)

**Ролі користувачів:**
1. Звичайний юзер:реєструватись, авторизуватись, створювати чати, надсилати повідомлення і файли , видаляти свої повідомлення, переглядати повідомлення.
2. Модератор чату: має всі можливості звичайного юзера та видалення повідомлень інших учасників, блокування користувачів у чаті.
3. Адміністратор: має доступ до системи (перегляд логів, керування ролями інших користуввачів, перегляд статистики сервера(кількість онлайн-користувачів, повідомлень, зареєстрованих користувачів, тд))

**Приклад деяких шляхів використання:**

Надсилання повідомлення
1. Користувач відкриває чат
2. Вводить повідомлення
3. Клієнт формує мережевий пакет
4. Пакет передається серверу
5. Сервер пересилає повідомлення іншому клієнту
6. Повідомлення відображається у реальному часі

Блокування користувача адміністратором
1. Адміністратор відкриває панель керування
2. Обирає користувача
3. Надсилає команду блокування
4. Сервер змінює статус користувача у базі даних
5. Заблокований користувач втрачає доступ до системи

Створення групового чату
1. Користувач створює новий чат
2. Обирає учасників
3. Сервер створює запис у базі даних
4. Учасники отримують повідомлення про створення чату

**Що можна/варто реалізовувати?**

База даних, клієнтська частина та серверна частина.

Клієнтська частина: графічний інтерфейс, формування пакетів з повідомлень, відображення повідомлень, взаємодія з сервером через мережу.

Серверна частина: обробка підключень користувачів до сервера, авторизація користувачів, маршрутизація повідомлень, робота з БД, логування.
